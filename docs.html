Kirill API Documentation

Base URL

http://localhost:8080/api/v1

Endpoints

POST /auth/register

Регистрация пользователя

{
  "name": "Ivan Petrov",
  "email": "ivan@example.com",
  "password": "pass123",
  "referralCode": "REF123",  // optional
  "avatarUrl": "https://..."  // optional
}

Response (200):

{
  "status": "success",
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    "name": "Ivan Petrov",
    "email": "ivan@example.com",
    "balance": 0,
    "createdAt": "2026-04-15T12:00:00.000Z",
    "updatedAt": "2026-04-15T12:00:00.000Z"
  }
}
POST /auth/login

Вход в систему

{
  "email": "ivan@example.com",
  "password": "pass123"
}

Response (200):

{
  "status": "success",
  "data": {
    "_id": "507f1f77bcf86cd799439011"
  }
}
POST /auth/profile

Получение профиля пользователя

{
  "userId": "507f1f77bcf86cd799439011"
}

Response (200):

{
  "status": "success",
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    "name": "Ivan Petrov",
    "email": "ivan@example.com",
    "balance": 100.50,
    "createdAt": "2026-04-15T12:00:00.000Z",
    "updatedAt": "2026-04-15T12:05:00.000Z"
  }
}
POST /billing/deposit

Пополнение баланса

{
  "userId": "507f1f77bcf86cd799439011",
  "amount": 100.50,
  "description": "Deposit"  // optional
}

Response (200):

{
  "status": "success",
  "data": {
    "transaction": {
      "_id": "507f1f77bcf86cd799439012",
      "userId": "507f1f77bcf86cd799439011",
      "amount": 100.50,
      "type": "deposit",
      "description": "Deposit",
      "createdAt": "2026-04-15T12:05:00.000Z"
    },
    "newBalance": 100.50
  }
}
GET /health

Health check

Response (200): OK

Error Response

{
  "status": "error",
  "errorMessage": "Error message"
}

Status Codes

200 - OK
400 - Bad Request
401 - Unauthorized
404 - Not Found
409 - Conflict
500 - Internal Server Error

Data Models

User

_id: string (ObjectId)
name: string
email: string (unique)
passwordHash: string
referralCode: string (optional)
avatarUrl: string (optional)
balance: number (default: 0)
createdAt: datetime
updatedAt: datetime

Transaction

_id: string (ObjectId)
userId: string (ObjectId)
amount: number
type: "deposit" | "withdrawal" | "payment"
description: string (optional)
createdAt: datetime
